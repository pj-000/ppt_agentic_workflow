## 输出要求
只输出本页的代码片段，用 <code> 标签包裹。
代码以 `{` 开头，以 `}` 结尾，内部第一行是 `let slide = pres.addSlide();`。
不要包含 require、new pptxgen()、pres.layout、writeFile。
本页必须承担清晰的信息任务，并为整份 deck 增加新的信息价值，不要重复上一页的开场和要点。
如果本页是 toc，必须清楚列出后续章节；如果本页是 closing，必须回扣前文并形成收束，不要只写致谢。
严格遵守视觉母题的配色和字体，保持与其他页一致。
布局仍然由你自主规划，不要套死模板；但装饰元素必须服从安全区规则，不能压住标题或正文。
页面必须有一个明确视觉重心，不允许把所有元素平均摆满整页。
优先做“标题层级 + 主视觉/主数字/主卡片 + 次级支持信息”的三层结构，避免 bullet wall。
如果正文里需要出现英文引号或单引号，请在 JS 字符串中写成 `\u0022` / `\u0027`，不要输出裸引号。
所有视觉元素必须是稳定可渲染的：本地图片、图表、形状、文本。禁止伪造图片路径和远程图片。

## 画布尺寸（必须遵守）
画布为 LAYOUT_WIDE：13.33" × 7.5"（英寸）。
- 背景色块/形状必须覆盖整个画布（x:0, y:0, w:13.33, h:7.5）
- 内容区域使用 0.5" 外边距（x 从 0.5 开始，最大宽度 12.33"）
- 不要把所有元素挤在画布中间一小块区域
- 封面/结尾页的标题和装饰必须铺满全屏，不能只占中间一小块

## 装饰安全区（必须遵守）
- 左侧深色竖条如果出现，默认贴左边或距离左边不超过 0.25"，宽度保持在 0.08"-0.18"，不要伸进正文区
- 左上角数学水印如果出现，必须是浅色低对比装饰，并放在标题上方或左上角留白区，不能与标题文本重叠
- 标题框必须避开左侧装饰和水印；当使用左侧竖条时，正文页标题建议从 x>=0.7" 开始
- 装饰元素优先让位于内容，可减少、缩小或移除装饰，不能为了装饰牺牲可读性

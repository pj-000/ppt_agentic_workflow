# 给老师汇报用精简版

## 1. 一句话方向

> 老师，我准备把项目方向从“一次性 PPT 生成”调整为“交互式 PPT 局部修改 Agent”。核心不是生成整套 PPT，而是当用户提出修改请求时，系统通过多粒度 Revision Index 定位应该修改哪些页面和元素，并规划最小化的局部编辑操作。

---

## 2. 目标会议

> 目标会议建议定为 **ACM IUI 2027**，CCF-B，人机交互与普适计算方向。

原因：

```text
1. 工作核心是人机交互式修改；
2. 系统使用智能 Agent/LLM；
3. 重点是用户控制、局部修改、多轮交互；
4. IUI 接受 intelligent user interfaces、human-agent interaction、user control、benchmarks、user studies 等方向。
```

---

## 3. 为什么不继续主打 PPT 生成

可以这样说：

> 直接做 PPT 生成有两个问题。第一，现有通用 PPT Skill 已经能完成很多底层生成能力，单纯封装生成流程创新不够。第二，我现在完整 PPT 生成流程不适合开源，不适合作为论文核心。因此我把生成后端降级为可替换执行器，把研究重点放在交互式局部修改和 Revision Index 上。

---

## 4. 核心问题

> 用户修改 PPT 时经常不是要重生成整套 PPT，而是说“这页太满了”“这里加个例子”“把刚才新增的那页讲简单点”。系统要解决的问题是：如何理解用户到底想改哪里、改哪些页面/元素、用什么操作修改，并且不破坏无关页面。

---

## 5. 研究定位

| 问题 | 回答 |
|---|---|
| 是 Agent 吗？ | 是，属于交互式 PPT 修改 Agent。 |
| 是算法创新吗？ | 有算法化模块，核心是 Index-guided localization and edit planning。 |
| 需要训练模型吗？ | 不需要，是 training-free 方法。 |
| 创新在哪里？ | Revision Index、页面/元素定位、编辑操作规划、动态 index 更新。 |
| PPT 生成后端怎么办？ | 作为黑盒 backend adapter，不作为论文核心。 |

汇报句：

> 这个工作不是训练新模型，而是一个 training-free 的 Agentic IUI framework。模型只作为意图理解和文本生成工具，PPT 后端只作为执行器，核心贡献是多粒度 Revision Index 和基于 Index 的局部修改规划。

---

## 6. 核心创新点

### 创新点 1：交互式 PPT 局部修改任务

> 与一次性生成整套 PPT 不同，我关注用户在已有 PPT 上进行多轮局部修改的场景。系统需要根据用户交互意图定位受影响页面/元素，并执行最小化修改。

### 创新点 2：多粒度 Revision Index

> Index 包含交互上下文、页面语义、页面元素、页面依赖和编辑操作五层，不只是找第几页，还能找元素、判断怎么改、支持多轮更新。

### 创新点 3：Index-guided 页面/元素定位

> 系统融合显式指代、当前选中页面、语义相似度、元素相关性、页面依赖和历史交互，定位受影响页面和元素。

### 创新点 4：Index-guided 编辑规划与动态更新

> 定位后，系统生成编辑操作，例如重写文本、加例子、拆页、合并页、调整顺序。修改完成后更新 Index，支持下一轮“刚才那页”等指代。

---

## 7. 系统流程

```text
用户交互请求
    ↓
意图解析
    ↓
Revision Index 查询
    ↓
页面/元素定位
    ↓
编辑操作规划
    ↓
局部修改执行
    ↓
修改结果验证
    ↓
Index 更新
    ↓
返回修改后 PPT 和解释
```

## 8. 数据集与实验

计划构建：

> **SlideReviseBench-ZH**

数据规模：

```text
40–60 套 PPT
500 条单轮修改任务
50–80 组多轮修改任务
```

任务类型：

```text
加例子、扩展内容、压缩内容、拆页、合并页、删除内容、调整顺序、改难度、改风格、加图示、全局统一、多轮引用。
```

实验：

```text
1. 页面/元素定位准确率；
2. 编辑操作预测准确率；
3. 局部修改质量；
4. 多轮交互一致性；
5. 用户研究。
```

Baselines：

```text
Keyword Search
Embedding Retrieval
LLM Direct Locate
Full Regeneration
LLM Direct Revision
Ours
```

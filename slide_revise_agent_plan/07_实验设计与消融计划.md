# 实验设计与消融计划

## 1. 实验总目标

实验要证明：

> **多粒度 Revision Index 能让交互式 PPT 局部修改更准确、更可控、更低干扰，并支持多轮交互。**

SlideReviseBench-ZH 主要用于 offline computational evaluation，包括 slide localization accuracy、element localization accuracy、operation prediction accuracy、local revision quality 和 multi-turn consistency。用户研究单独评估 human-centric outcomes，包括 task completion time、number of turns、manual corrections、perceived control、trust、satisfaction、preference 和 qualitative feedback。

不要只展示 case study。必须有：

```text
1. 页面/元素定位实验；
2. 编辑操作预测实验；
3. 局部修改质量实验；
4. 多轮交互一致性实验；
5. 消融实验；
6. 用户研究。
```

## 2. 实验 1：页面 / 元素定位准确率

### 任务

给定：

```text
PPT + Revision Index + 用户修改请求 + 交互上下文
```

预测：

```text
应该修改哪些页面和元素
```

### Baselines

| 方法 | 描述 |
|---|---|
| Keyword Search | 用关键词匹配 slide 文本 |
| Embedding Retrieval | 指令和 slide 文本做向量相似度 |
| LLM Direct Locate | 直接让 LLM 判断该改哪页 |
| Selection-only | 只使用当前选中页面 |
| Outline-only | 只使用标题/大纲结构 |
| Ours | 完整 Revision Index |

### 指标

| 指标 | 含义 |
|---|---|
| Slide Top-1 Accuracy | 排名第一页面是否命中 gold |
| Slide Top-3 Recall | gold 页面是否在前三 |
| Slide Precision / Recall / F1 | 多页定位 |
| Element Accuracy | 元素定位是否正确 |
| Element F1 | 多元素定位 |
| MRR | 正确目标平均倒数排名 |
| Unintended Target Rate | 是否错误定位无关页面 |

### 预期结果

Ours 应该在多页定位、元素定位和含指代的任务上明显优于 keyword、embedding 和 LLM Direct。

## 3. 实验 2：编辑操作预测准确率

### 任务

给定：

```text
用户修改请求 + interaction context + localization result + deck index
```

预测：

```text
structured gold operation objects
```

### Baselines

| 方法 | 描述 |
|---|---|
| LLM Direct Plan | 直接让 LLM 输出操作 |
| Rule-only Planner | 只用简单规则 |
| No Element Index | 去掉元素信息 |
| No Dependency Index | 去掉页面依赖 |
| No Operation Index | 去掉操作索引 |
| Full Ours | 完整方法 |

### 指标

```text
Operation Accuracy
Macro-F1
Micro-F1
Exact Match
Human Accept Rate
```

### 预期结果

Ours 在 SPLIT_SLIDE、add_example 意图对应的 ADD_SLIDE/ADD_TEXT_BLOCK、DELETE_TEXT_BLOCK、CHANGE_STYLE 等复杂规划场景上应更稳定。

## 4. 实验 3：局部修改质量

### 任务

执行 edit plan 后，评估修改后的 PPT 是否满足用户意图，同时是否尽量少影响无关页面。

### Baselines

| 方法 | 描述 |
|---|---|
| Full Regeneration | 整套 PPT 重新生成 |
| LLM Direct Revision | 直接让 LLM 改 PPT |
| Keyword-based Revision | 关键词定位后局部改 |
| Ours | Index-guided Local Revision |

### 指标

| 指标 | 含义 |
|---|---|
| Task Success Rate | 用户修改需求是否完成 |
| Minimal Change Score | 是否只改必要页面 |
| Unaffected Slide Preservation | 未受影响页面是否保持 |
| Visual Consistency | 修改页和原 PPT 风格是否一致 |
| Deck Coherence | 修改后整体是否连贯 |
| Human Preference | 人类偏好 |

### Minimal Change Score 建议

```text
MinimalChange = 1 - (#unexpected_modified_slides / #total_slides)
```

也可以结合元素级 diff：

```text
ElementPreservation = unchanged_unaffected_elements / total_unaffected_elements
```

## 5. 实验 4：多轮交互一致性

### 任务示例

```text
Turn 1: 给 Transformer 部分加一页例子。
Turn 2: 把刚才新增的那页讲简单一点。
Turn 3: 把这页移到公式页之前。
```

### 指标

| 指标 | 含义 |
|---|---|
| Multi-turn Success Rate | 多轮任务整体完成率 |
| Reference Resolution Accuracy | “刚才那页”等引用解析准确率 |
| Index Consistency | 修改后 index 是否正确 |
| Error Accumulation Rate | 多轮后错误是否累积 |
| Average Turns to Completion | 完成任务平均交互轮数 |

### 预期结果

带 Dynamic Index Update 的完整方法应优于 w/o Dynamic Update。

## 6. 消融实验

| 版本 | 去掉内容 | 验证问题 |
|---|---|---|
| Full | 完整方法 | 主结果 |
| w/o Context Index | 去掉交互上下文 | “这页/这里/刚才那页”是否下降 |
| w/o Element Index | 去掉元素索引 | 元素级定位是否下降 |
| w/o Dependency Index | 去掉页面依赖 | 目录/总结/后续页同步是否下降 |
| w/o Operation Index | 去掉操作索引 | 操作预测是否下降 |
| w/o Dynamic Update | 去掉动态更新 | 多轮性能是否下降 |

## 7. 用户研究设计

IUI 需要 human-centric evidence，建议必须做。

### 参与者

```text
12–18 名参与者
```

建议构成：

```text
- 研究生 8–12 人
- 助教/教师 2–3 人
- 有 PPT 制作经验的本科生 2–5 人
```

### 对比条件

建议两个条件：

```text
A. Baseline：普通聊天式 LLM 修改 / 直接重生成
B. Ours：SlideReviseAgent
```

如果时间足够，加第三个：

```text
C. 手动 PowerPoint 修改
```

### 用户任务

每个参与者完成 3 个任务：

```text
1. 给某页增加例子；
2. 将一页拆成两页并保持风格；
3. 调整某部分难度或顺序。
```

建议 within-subject design，任务顺序随机化。

### 测量指标

| 指标 | 方法 |
|---|---|
| Completion Time | 系统记录 |
| Number of Turns | 系统记录 |
| Manual Corrections | 用户完成后记录 |
| Task Success | 人工判断 |
| Perceived Control | 5/7 点 Likert |
| Trust | 5/7 点 Likert |
| Satisfaction | 5/7 点 Likert |
| Preference | 选择 A/B/C |
| Qualitative Feedback | 半结构访谈 |

### 问卷问题示例

```text
1. 我觉得系统准确理解了我的修改意图。
2. 我觉得系统修改了正确的页面。
3. 我觉得系统没有破坏无关页面。
4. 我能理解系统为什么这样修改。
5. 我觉得这个系统比直接重生成 PPT 更可控。
6. 我愿意在真实 PPT 制作中使用这个系统。
```

## 8. 统计分析

### 离线实验

```text
- 平均值 + 标准差；
- bootstrap confidence interval；
- paired significance test。
```

### 用户研究

根据数据分布选择：

```text
- paired t-test；
- Wilcoxon signed-rank test；
- effect size；
- qualitative coding。
```

### 标注一致性

对于 benchmark gold label：

```text
- Cohen's kappa：两个标注者；
- Fleiss' kappa：多个标注者；
- 或者用 agreement rate。
```

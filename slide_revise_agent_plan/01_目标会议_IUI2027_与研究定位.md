# 目标会议：ACM IUI 2027 与研究定位

## 1. 目标会议

建议目标会议定为：

> **ACM International Conference on Intelligent User Interfaces, IUI 2027**

定位：

```text
CCF-B
人机交互与普适计算方向
目标论文类型：Full Paper / Regular Paper
```

选择原因：本项目最终方向是“交互式 PPT 局部修改 Agent”，它同时涉及：

```text
- Human-AI interaction
- Intelligent user interfaces for generative AI
- User control and steering of LLMs/agents
- Human-agent interaction
- Evaluation of intelligent user interfaces
- Benchmarks, datasets and reproducibility
```

这些都与 IUI 主题高度一致。

## 3. 为什么不是 CIKM / ICME / EMNLP

### CIKM

CIKM 适合信息检索、知识管理、indexing、benchmark。如果你最后把工作写成“PPT 页面/元素检索方法”，CIKM 也可行。

但你的工作包含明显的人机交互：

```text
用户修改请求
当前选中页面/元素
多轮交互
用户控制感
局部修改体验
用户研究
```

这些更适合 IUI。

### ICME / ICMR

如果你把重点放在 PPT 页面视觉理解、版式修复、图文布局一致性，可以考虑 ICME/ICMR。但目前核心是“交互修改 + index 定位 + edit planning”，不是多媒体视觉生成。

### EMNLP / COLING

如果你把核心改成文档到 SlideIR 的结构化生成，可以考虑 NLP 会议。但当前方向不训练模型，也不是纯语言生成任务。

---

## 4. 最终研究定位

本项目应定位为：

> **Training-free Agentic IUI System + Multi-granularity Revision Index + Interactive Revision Benchmark**

中文表述：

> **一个不训练新模型的交互式智能用户界面系统，核心方法是多粒度 Revision Index，用于支持 PPT 局部修改中的页面/元素定位和编辑操作规划。**

---

## 5. 和当前仓库的关系

当前仓库 `directionai-agent-backend` 的公开 README 显示，它已经有独立 PPT 生成后端，包括 CLI、FastAPI 服务、两阶段 outline/from-outline 生成、文档上传、PPT 预览和评估接口。后续新方向不要推翻这些能力，而是：

```text
1. 保留 ppt_backend/ 和 runtime/ 作为已有后端能力；
2. 新增 slide_revise_agent/ 作为论文研究模块；
3. 用 backend_adapter.py 调用现有生成/修改能力；
4. 另做 lightweight executor 保障可复现；
5. 论文核心只依赖 index、localization、edit planning 和 benchmark。
```

---

## 6. 投稿主张

IUI 论文主张建议写成：

> Slide authoring is iterative rather than one-shot. Existing slide generation tools can produce decks, but users often need controllable local revisions. We propose SlideReviseAgent, a training-free interactive revision framework that builds a Multi-granularity Revision Index over interaction contexts, slide semantics, slide elements, inter-slide dependencies, and edit operations. The index enables accurate localization of affected slides and elements, index-guided edit planning, and dynamic index updates for multi-turn revisions.

中文汇报版：

> PPT 创作本质上是反复修改，而不是一次性生成。现有 PPT 生成系统生成整套 PPT 后，用户很难控制局部修改，也容易因为重新生成破坏原有页面。我的工作提出一个基于多粒度 Revision Index 的交互式 PPT 局部修改 Agent，使系统能根据用户交互意图定位到具体页面和元素，并生成最小化的局部修改计划。

## 7. IUI 对论文的硬要求转化成项目要求

| IUI 要求 | 项目对应动作 |
|---|---|
| AI + HCI 交叉 | 系统是 LLM/Agent 驱动的 PPT 修改界面 |
| computational evidence | 做页面定位、元素定位、操作预测、消融实验 |
| human-centric evidence | 做 12–18 人用户研究 |
| rigorous evaluation | benchmark + offline metrics + user study |
| double-blind | 论文、补充材料、代码仓库匿名化 |
| GenAI disclosure | 论文 references 前加 GenAI Usage Disclosure |
| human subjects ethics | 用户研究前咨询导师/学院伦理要求 |
| supplemental encouraged | 准备 demo video、问卷、数据说明、匿名代码 |

---

## 8. 最终结论

目标会议确定为：

> **ACM IUI 2027**

论文方向确定为：

> **SlideReviseAgent：基于多粒度 Revision Index 的交互式 PPT 局部修改 Agent。**

核心创新不是 PPT 生成，也不是模型训练，而是：

```text
1. 交互式 PPT 局部修改任务；
2. 多粒度 Revision Index；
3. Index-guided 页面/元素定位；
4. Index-guided 编辑操作规划；
5. 动态 Index 更新；
6. SlideReviseBench-ZH + 用户研究。
```
